# 🦜 LangChain Master Codebook
A comprehensive technical repository documenting my complete journey through the **CampusX Generative AI with LangChain** series. This repository serves as a structured guide for building modern LLM applications, from foundational components to advanced RAG pipelines.

## 🚀 Key Modules & Implementations
This codebook covers the end-to-end lifecycle of LangChain development:

* **Model Integration:** Working with ChatModels (Gemini, OpenAI) and managing system/human message flows.
* **Document Loading & Ingestion:** Techniques for loading data from PDFs, CSVs, and Web sources.
* **Advanced Text Splitting:** Implementing Recursive and Semantic chunking strategies for optimal context retrieval.
* **Vector Databases:** Using **FAISS** for efficient similarity search and persistent vector storage.
* **Retrieval-Augmented Generation (RAG):** Building pipelines that allow LLMs to answer questions based on custom datasets.
* **LCEL (LangChain Expression Language):** Creating modular and chainable logic using the latest LangChain syntax.
* **Output Parsing:** Forcing LLMs to return structured data like JSON or specific Pydantic schemas.
* **Agents & Tools:** Implementing ReAct agents capable of using external tools (Search, Math) to solve complex tasks.

## 🛠️ Tech Stack
* **Framework:** LangChain (2026 Modular Version)
* **LLMs:** Google Gemini 1.5/3.0 & OpenAI GPT-4o
* **Embeddings:** Google Generative AI Embeddings
* **Vector Store:** FAISS
* **Environment:** Python 3.12, VS Code, and Google Colab

## 📂 Repository Organization
The repository is structured to mirror the LangChain ecosystem:
* `ChatModels/` & `EmbeddedModels/` - Core model interfaces.
* `LANGCHAIN_DOCUMENT_LOADERS/` - Data ingestion scripts.
* `LANGCHAIN_PROMPTS/` - Prompt engineering templates.
* `notebook/` - Comprehensive Jupyter notebooks for RAG and Agentic workflows.
* `requirements.txt` - Complete list of dependencies for reproducibility.
